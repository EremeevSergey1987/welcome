Для удаления пакета используйте:

sudo apt-get --purge remove "имя_пакета" Чтобы удалить конфигурационные файлы можно использовать опцию --purge или команду purge

sudo apt-get --purge "имя_пакета"

ls -l - выводит подробную информацию о файлах и папках
ls -la - выводит все файлы и папки которые есть в директории
ll - псевдоним для bash
cat - позволяет нам прочитать файл но только если файл не большой
more - древняя команда для чтения, она позволяет просматривать файл частями.
less - это противоположность команды more. Она позволяет двигаться по файлу как вверх так и вниз
tail - позволяет просмотреть последние 10 строк файла.
tail -n 100 позволяет просмотреть последние 100 строк файла.
tail -f позволяет выводить в реальном времени - то что дописывается в файл.
php logger.php - выводит 10 случайных логов
php logger.php >> my_log.php & - записывать логи в свой файл. А амперсант говорит о том что программу можно запустить в фоновом режиме.
Чтобы выйти из режима просмотра нужно нажать ctrl + c

mkdir test_dir - команда для создания директории
touch readme.txt - команда для создания файла
rmdir test_dir - удаляем папку test_dir
rm readme.txt - удалить файл можно командой rm
cp index.html index2.html - копируем файл
cp -R test test2 - копирование папки с файлами. Рекурсия для того чтобы все файлы скопировались
rm -r - удалить директорию с вложенными файлами. Рекурсия для удаления вложенных файлов.
rm -r log* - удаляем все что начинается со слова "log"
Файлы удаляются в linux безвозвратно и удалять нужно очень осторожно
mv index.html index2.html - переименовать файл
mc - вызов редактора для редактирования и сохранения файлов и папок
sudo mcedit index2.html - вызов редактора mc напрямую
sudo nano index2.html - встроенный редактор файлов
vi - редактор самый старый редактор и он не удобный
cat index.html index2.html - выведет содержимое обоих файлов.
cat index.html index2.html > total.php создаст единый файл.

df - позволяет посмотреть скольок места осталось на диске
df -h - позволяет посмотреть скольок места осталось на диске в более удобном виде
du -h - просмотреть разеры папок в мегабайтах (Можно указывать маску тех файлов и папок по которым нужно узнать размер ./* )

std out - это первый поток 1.
std in - это поток ввода. Это нулевой поток 0 !
std error - это поток ошибок он идет под номером 2

Файл записывать можно каждый раз с нуля. Используя одинарную угловую скобку >
Либо можно дописывать данные в его конец используя две скобки >>

wc (имя файла) - пишет количество строк, количество слов и количество символов в строке.
ls | wc - перенаправляем потока данных с помощью конвеера, пайп лайна, вертикальной черты...
wc < ls.log - закинуть файл ls.log команде wc
mysql (dbname) < damp.sql - заливаем дамп базы в базу данных

&& - Двойной амперсанд означает, что следующая команда выполнится после завершения предыдущей.
